Tennis Match Data Definitions

Tournament Related

Field Description
tourney_id A unique identifier for each tournament, such as 2020-888. The exact formats are borrowed from several different sources, so while the first four characters are always the year, the rest of the ID doesn't follow a predictable structure.
tourney_name Self-explanatory. Such as : 'Adelaide', 'Doha', 'Chennai', 'Australian Open', 'US Open', or 'Wimbledon'
surface Self-explanatory in between Hard, Carpet, Clay, or Grass
draw_size Number of players in the draw, often rounded up to the nearest power of 2. (For instance, a tournament with 28 players may be shown as 32.)
tourney_level For men: 'G' = Grand Slams, 'M' = Masters 1000s, 'A' = other tour-level events, 'C' = Challengers, 'S' = Satellites/ITFs, 'F' = Tour finals and other season-ending events, and 'D' = Davis Cup.
For women: Additional codes include 'P' = Premier, 'PM' = Premier Mandatory, 'I' = International. ITF levels are given by prize money, such as '15' = ITF $15,000. Older WTA codes include 'T1' for Tier I, and 'D' for Federation/Fed/Billie Jean King Cup.
Others: 'E' = exhibition, 'J' = juniors, 'T' = team tennis (not yet in the dataset).
tourney_date Eight digits, YYYYMMDD, usually the Monday of the tournament week.
match_num A match-specific identifier. Often starting from 1, sometimes counting down from 300, and sometimes arbitrary.

Player Related

Note: The values for player-related fields appear for both the winner and loser, with suffixes 'winner_' for player=winner and 'loser_' for player=loser.

Field Description
player_id The player_id used in this repo for the player of the match.
player_seed Self-explanatory.
player_entry 'WC' = wild card, 'Q' = qualifier, 'LL' = lucky loser, 'PR' = protected ranking, 'ALT'= alternates, 'ITF' = ITF entry, and there are a few others that are occasionally used.
player_name Self-explanatory.
player_hand R = right, L = left, U = unknown. For ambidextrous players, this is their serving hand.
player_ht Height in centimeters, where available.
player_ioc Three-character country code.
player_age Age, in years, as of the tourney_date.
player_rank Player's ATP or WTA rank, as of the tourney_date, or the most recent ranking date before the tourney_date.
player_rank_points Number of ranking points, where available.

Performance Stats

Note: The values for player-related fields appear for both the winner and loser, with suffixes 'w_' for winner and 'l_' for loser.

Field Description
p_ace Player's number of aces.
p_df Player's number of double faults.
p_svpt Player's number of serve points.
p_1stIn Player's number of first serves made.
p_1stWon Player's number of first-serve points won.
p_2ndWon Player's number of second-serve points won.
p_SvGms Player's number of serve games.
p_bpSaved Player's number of break points saved.
p_bpFaced Player's number of break points faced.

Match Related

Field Description
score Self-explanatory.
best_of '3' or '5', indicating the number of sets for this match.
round Stage of the match within the tournament: with Finals such as : 'QF', 'SF', 'F', initial rounds marked as : 'R16', 'R32', 'R64', 'R128', and qualifiers such as : 'Q1', 'Q2', 'Q3', Round Robin - 'RR', Backhand Rally / Informal Match -'BR', EarlyRound-'ER', Finals - 'f']
minutes Match length, where available.
